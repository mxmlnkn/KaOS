Vor einer Datenuebertragung muss der UART zunaechst initialisiert werden,
indem die einzelnen Steuerregister entsprechend den gewuenschten
uebertragungsparametern programmiert werden. Die eigentliche serielle
uebertragung erfolgt meist in kleinen Bloecken von acht Bits, also als
einzelne Bytes. Um ein Byte zu senden, schreibt man es in das Sende-Halteregister:
 
(Offset=0, schreiben)
D7    D6    D5    D4    D3    D2    D1    D0
Sende-Halteregister(transmitter holding register (THR)) enthaelt das
zu sendende Zeichen.Bit 0 wird zuerst gesendet.
 
(Offset=0, lesen)
D7    D6    D5    D4    D3    D2    D1    D0
Empfangs-Halteregister(receiver buffer register (RBR)) enthaelt das
zu empfangende Zeichen.
 
************************************************************************
 
Fuer die uebertragung muß man eine uebertragungsgeschwindigkeit waehlen,
die in Bits pro Sekunde (Baud) angegeben wird. Bei 1200 Baud werden
zusammen mit je einem Startbit und einem Stopbit in einer Sekunde bis zu
120 Bytes uebertragen. Der UART 8250 verwendet einen Quarz mit 1,8432 MHz.
Diese Frequenz wird intern durch einen einstellbaren Teilerfaktor zur
16-fachen Baudrate geteilt. Der erforderliche Teilerfaktor berechnet sich
also zu: Teilerfaktor = 1843200 /(16 x Baudrate ) = 115200 / Baudrate
 
(Offset=0, DLAB=1)
Baudratenregister, Lowbyte
D7    D6    D5    D4    D3    D2    D1    D0
 
(Offset=1, DLAB=1)
Baudratenregister, Highbyte
D15   D14   D13   D12   D11   D10   D9    D8

Von der seriellen Schnittstelle koennen Interrupts ausgeloest werden.
Dazu muß das Interrupt-Freigaberegister programmiert werden. Um einen
oder mehrere Interrupts zu aktivieren, muß das entsprechende Bit gesetzt
werden.
 
(Offset=1, DLAB=0)
Interrupt-Freigaberegister - interrupt enable register (IER)
7     6     5     4     3     2     1     0
0     0     0     0     SINP  ERBK  TBE   RxRD
 
Interruptfreigabe fuer:
SINP: Zustansaenderung einer Handshake-Leitung
ERBK: Fehler oder BREAK erkannt
TBE: Sende-Haltepuffer ist leer
RxRD: Ein Zeichen wurde empfangen
 
Ein Fehler kann beim Empfang eines Zeichens erkannt werden, wenn z.B.
die vereinbarte Zeichenlaenge oder die Anzahl der Stopbits nicht
eingehalten wird. Liegt am Eingang RXD ein anhaltender High-Pegel, dann
spricht man von einem BREAK-Zustand. Dieser Zustand wuerde nach Ablauf
der normalen uebertragungszeit der vereinbarten Datenbits wie ein
Empfangsfehler gewertet.
 
************************************************************************
 
Um Interrupts auszuloesen, muessen noch zwei weitere Bedingungen erfuellt
sein: Zum einen muß der Interruptcontroller entsprechend programmiert
      sein, zum anderen muß die Leitung OUT2 (Bit 3 im Modem-Steuerregister)
      gesetzt sein.
Wenn einer von mehreren moeglichen Interrupts aufgetreten ist, kann man
im Interrupt-Erkennungsregister die Quelle erfahren:
 
(Offset=2, lesen)
Interrupt-Erkennungsregister - interrupt identification register (IIR)
7     6     5     4     3     2     1     0
FON   FON2  0     0     ID2   ID1   ID0   PND
FON: FIFO eingeschaltet
FON2: FIFO eingeschaltet (nur 16550A)
ID2: Identifizierungsbit: Timeout
ID1, ID0: Identifizierungsbits:
00: Zustansaenderung einer Handshake-Leitung
01: Sende-Haltepuffer ist leer
10: Ein Zeichen wurde empfangen
11: Fehler oder BREAK-Bedingung erkannt
PND: Interrupt ist anhaengig
 
Ein abhaengiger Interrupt nach einer Zustandsaenderung einer
Handshakeleitung wird durch das Lesen des Modem-Statusregisters wieder
geloescht. Ebenso loescht ein Lesen des Empfangs-Halteregisters
einen Empfangs-Interrupt und das Lesen des Leitungs-Statusregisters
einen Fehler- oder BREAK-Interrupt. Der Interrupt durch ein leeres
Sende-Halteregister kann entweder durch das Lesen des Interrupt-
Erkennungsregisters oder durch Schreiben in das Sende-Halteregister
geloescht werden.
 
************************************************************************
 
Der FIFO-Speicher im 16550
Der UART 8250 und sein Nachfolgetyp 16450 verarbeiten jeweils nur ein
Zeichen. Wird ein neues Zeichen empfangen, bevor das alte ausgelesen wurde,
kommt es zum Datenverlust. Um auch bei hoeheren uebertragungsgeschw-
indigkeiten andere Aufgaben parallel durchfuehren zu koennen, setzt man
den neueren 16550 mit zwei 16-Byte Zwischenspeichern fuer die Sende- und
Empfangsrichtung ein. Die zusaetzlichen Speicher sind als sog. FIFO-
(first in first out) Speicher ausgelegt, so daß die ausgelesenen Zeichen
immer in der Reihenfolge des Empfangs erscheinen.
 
Der FIFO-Speicher ist nach einem Reset zunaechst inaktiv und muß bei
Bedarf durch das FIFO-Controllregister eingeschaltet werden. Dieses
Register ist nur in Schreibrichtung unter der Offsetadresse 2 zugaenglich,
beim Auslesen befindet sich hier das Interrupt-Erkennungregister, das
ebenfalls eine erweiterte Funktion hat:
 
(Offset=2, schreiben)
FIFO-Steuerregister - FIFO control register (FCR)
7     6     5     4     3     2     1     0
TL1   TL0   0     0     0     FRT   FRR   FEN
 
TL1, TL0: Triggerlevel, Interrupt nach:
 00: 1 Byte
 01: 4 Bytes
 10: 8 Bytes
 11: 14 Bytes
FRT: FIFO-Reset fuer Sender
FRR: FIFO-Reset fuer Empfaenger
FEN: FIFO einschalten
 
Will man ohne FIFO arbeiten und eine Kompatibilitaet zum 8050 erzwingen,
dann sollte der Wert 6 in das Register geschrieben werden. Damit erfolgt
zugleich ein Reset und ein Ausschalten des FIFO. Ob der FIFO-Modus
eingeschaltet wurde, laeßt sich an einem zusaetzlichen Statusbit des
Interrupt-Erkennungsregisters ablesen.
 
************************************************************************
 
Das Leitungs-Steuerregister braucht im Normalfall nur zur Initialisierung
der seriellen Schnittstelle beschrieben werden. Es enthaelt acht
Steuerbits zur Festlegung der uebertragungsparameter:
 
(Offset=3, DLAB=0)
Leitungs-Steuerregister - Line Control Register (LCR)
7     6     5     4     3     2     1     0
DLAB  BRK   PAR2 PAR1 PAR0    STOP  DAB1  DAB0
 
DLAB: Zugriff auf das Baudratenregister freigeben
BRK: BREAK-Zustand an TXD setzen
PAR2...PAR0: Festlegung des Paritaetsbits
 000: keins
 001: ungerade
 011: gerade
 101: mark
 111: space
STOP: Festlegung von zwei Stopbits
DAB1...DAB0: Anzahl der Datenbits
 00: 5 Bits
 01: 6 Bits
 10: 7 Bits
 11: 8 Bits
 
Das DLAB-Bits bewirkt eine interne Umschaltung der Register mit den
Offsetadressen 0 und 1. Es muß gesetzt werden, um die Baudrate
einzustellen, im normalen Betrieb muß es dagegen geloescht sein. Das
BRK-Bit kann verwendet werden, um den Zustand der Leitung TXD als Ausgang
voellig unabhaengig von der normalen Funktion des UARTs zu steuern.
 
Die uebrigen Bits des Registers legen fuer den Sender und den Empfaenger
des UARTs ein eventuelles Paritaetsbit, die Anzahl der Stopbits und die
Anzahl der Datenbits fest. Ein uebliches Format ist z.B. 8 Datenbits,
kein Paritaetsbit und ein Stopbit. Fuer diese Einstellung muessen nur die
beiden Bits DAB1 und DAB0 gesetzt werden (Registerinhalt = 3). Zusammen
mit dem Startbit werden dann insgesamt 10 Bits uebertragen, so daß z.B.
bei einer uebertragungsgeschwindigkeit von 19200 Baud genau 1920 Zeichen
pro Sekunde uebertragen werden koennen.
 
************************************************************************
 
Das Leitungs-Steuerregister dient im wesentlichen zum Steuern der
zusaetzlichen Ausgaenge des UART. Waehrend DTR und RTS an den
Schnittstellenstecker gefuehrt werden, sind OUT2 und OUT1 als zusaetzliche
Mehrzweckausgaenge beim PC nicht von außen zugaenglich. OUT2 erfuellt
aber eine wichtige Funktion beim Aktivieren von Interrupts. Diese Leitung
muß naemlich gesetzt werden, um ueberhaupt einen Interrupt weiterzuleiten.
 
(Offset=4)
Modem-Steuerregister - Modem Control Register (MCR)
7     6     5     4     3     2     1     0
0     0     0     LOOP  OUT2  OUT1  RTS   DTR
 
LOOP: Interne Rueckkopplung zu Testzwecken
OUT2: Leitung OUT2 setzen, um Interrupts zu ermoeglichen
OUT1: interne Leitung OUT1 setzen
RTS: Leitung RTS setzen (request-to-send)
DTR: Leitung DTR setzen (data-terminal-ready)
 
Das LOOP-Bit kann sinnvoll eingesetzt werden, um eigene Programme zu
testen. Die ausgesendeten Zeichen werden empfangen, ohne daß eine externe
Verbindung zwischen TXD und RXD hergestellt werden muß.
 
************************************************************************
 
Das Leitungs-Statusregister enthaelt sieben Statusbits, ueber die sich
aktuelle Zustaende des seriellen Senders und des seriellen Empfaengers
auslesen lassen.
 
(Offset=5)
Leitungs-Statusregister - Line Status Register (LSR)
7     6     5     4     3     2     1     0
0     TXE   TBE   BREK  FRMF  PARF  ueBLF RXRD
 
TXE: Letztes Zeichen ist vollstaendig ausgesendet
TBE: Sende-Halteregister ist leer
BREK: BREAK-Zustand an RXD erkannt
FRMF: Format-Fehler erkannt
RARF: Paritaetsfehler erkannt
ueBLF: ueberlauffehler erkannt
RXRD: Ein Zeichen wurde empfangen
 
Sobald das TBE-Bit gesetzt ist, darf ein neues Zeichen in das
Sende-Haltegegister geschrieben werden. Dieses Zeichen wird aber erst
dann in das Schieberegister des seriellen Senders uebernommen, wenn kein
anderes Zeichen mehr bearbeitet werden muß. Sobald das zuletzt in das
Sende-Halteregister geschriebene Zeichen in das Schieberegister uebernommen
wird, setzt der UART sein TBE-Bit. Das TXE-Bit wird dagegen erst dann
gesetzt, wenn das letzte Zeichen vollstaendig uebertragen wurde.
 
Fuer die Empfangsrichtung zeigt das RXRD-Bit, daß ein Zeichen zur
Verfuegung steht. Es muß rechtzeitig aus dem Empfangs-Halteregister
ausgelesen werden, um nicht von einem folgenden Zeichen ueberschrieben
zu werden. Im letzteren Fall wuerde durch das ueBLF-Bit ein ueberlauf
angezeigt. ueberlaeufe lassen sich verhindern, indem entweder durch ein
genuegend schnelles "Polling" das RXRD-Bit laufend beobachtet wird, oder
indem eine Interruptroutine benutzt wird, die ein ankommendes Zeichen
verarbeitet.
 
************************************************************************
 
Der Zustand der Eingaenge DCD, RI, DSR und CTS kann ueber das Modem-
Statusregister gelesen werden. Waehrend die oberen vier Bits den Zustand
der vier Leitungen direkt widerspiegeln, zeigen die unteren vier Bits
an, ob seit dem letzten Lesezugriff auf das Modem-Statusregister eine
Zustandsaenderung an einer Leitung aufgetreten ist.
 
(Offset=6)
Modem-Statusregister - Modem Status Register (MSR)
7     6     5     4     3     2     1     0
DCD   RI    DSR   CTS   DDCD  DRI   DDSR  DCTS
 
DCD: Leitung DCD ist gesetzt (data carrier detect)
RI: Leitung RI ist gesetzt (ring indicator)
DSR: Leitung DSR ist gesetzt (data set ready)
CTS: Leitung CTS ist gesetzt (clear to send)
DDCD: Zustandsaenderung an DCD (delta data carrier detect)
DRI: Zustandsaenderung an RI (trailing edge ring indicator)
DDSR: Zustandsaenderung an DSR (delta data set ready)
DCTS: Zustandsaenderung an CTS (delta clear to send)
 
Notes:      bits 0-3 are reset when the CPU reads the MSR
            bit 4 is the Modem Control Register RTS during loopback test
            bit 5 is the Modem Control Register DTR during loopback test
            bit 6 is the Modem Control Register OUT1 during loopback test
            bit 7 is the Modem Control Register OUT2 during loopback test
